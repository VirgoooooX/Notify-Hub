# 安全与可靠性

## 1. 威胁模型

Notify Hub 会同时持有：

- 企业微信应用 Secret；
- 外部 API Client Key；
- 插件访问外部数据源的 Token/Cookie；
- 企业微信成员身份；
- 提醒和通知内容；
- 可能包含隐私的语音、图片和日志。

同时它还具备：

- 向企业微信成员发送消息；
- 定时运行插件；
- 后续执行受控脚本；
- 接收公网 Webhook。

因此默认按“高价值家庭基础设施服务”处理，而不是普通小脚本。

## 2. 身份认证

### 管理员

- 使用强密码哈希；
- 登录失败限流；
- 会话 Token 短期有效；
- Refresh Token 可撤销；
- 管理员接口与外部 API Client 接口使用不同认证方式；
- 生产环境只允许 HTTPS；
- 初始管理员创建后关闭初始化入口；
- 后续可选 TOTP，但不作为 MVP 阻塞项。

### API Client

- 每个来源一个独立 Key；
- Key 只显示一次；
- 数据库只保存哈希；
- 保存可识别前缀；
- 支持轮换和吊销；
- 支持允许的事件类型、接收人和优先级；
- 默认禁止广播；
- 记录最后使用时间但不记录完整 Key。

### 企业微信回调

- 不使用 API Key；
- 必须验证签名并解密；
- 校验时间戳与重放窗口；
- 回调重复幂等；
- 回调 URL 不暴露调试信息。

## 3. Secret 管理

Secret 分类：

- 系统 Secret：数据库加密主密钥、JWT/Session Key；
- 渠道 Secret：企业微信 Secret、Token、AES Key；
- 插件 Secret：X API Token、RSS 凭据、Cookie；
- Client Secret：外部 API Key。

规则：

1. `.env.example` 只放占位符；
2. 真实 `.env` 进入 `.gitignore`；
3. 数据库存储 Secret 时使用认证加密；
4. 主加密密钥来自环境变量或 Secret 文件；
5. 前端永不回显 Secret；
6. API 只返回 `configured: true/false`；
7. 日志过滤常见敏感 Header 和字段；
8. 导出配置时默认排除 Secret；
9. 支持 Secret 轮换；
10. 启动时检测默认或弱密钥并拒绝生产模式启动。

## 4. 权限模型

第一阶段至少区分：

- `admin`：管理平台全部资源；
- `api_client`：受限提交事件；
- `wecom_member`：管理自己的提醒；
- `plugin`：使用声明并授权的 Context 能力。

敏感权限：

- 广播到 `@all`；
- 通知任意接收人；
- 发送 critical；
- 发送语音；
- 读取局域网地址；
- 访问插件 Secret；
- 运行脚本；
- 修改插件配置。

权限判断必须在服务端完成，不能只依赖前端隐藏按钮。

## 5. 插件安全

第一阶段只运行可信插件。即便如此也要限制：

- 只通过 PluginContext 访问平台；
- 不提供数据库连接；
- 不提供渠道 Client；
- 不提供其他插件 Secret；
- HTTP Client 限制目标和响应大小；
- 明确超时；
- 最大并发；
- 连续失败熔断；
- 禁止动态 `pip install`；
- 代码评审禁止 `eval`、`exec`、`shell=True`；
- 插件目录不可由 Web 请求直接写入。

未来第三方插件必须迁移到子进程或容器沙箱后再开放安装能力。

## 6. SSRF 和网络访问

通用 HTTP 监控或插件网络请求可能被滥用访问内网。

默认阻止：

- `127.0.0.0/8`；
- `169.254.0.0/16`；
- IPv6 loopback/link-local；
- 云元数据地址；
- `file://`、`gopher://` 等非 HTTP 协议；
- 重定向到禁止地址；
- 未验证 TLS。

私有 NAS 监控确实需要访问局域网时，由管理员按插件显式授权 CIDR 或主机名。

DNS 解析和重定向后都要重新校验，避免 DNS rebinding。

## 7. 输入和媒体安全

- JSON 请求体限制大小；
- title/content 长度限制；
- URL 协议白名单；
- HTML 默认按纯文本处理；
- 前端展示时转义；
- 图片和音频验证真实 MIME，不只看扩展名；
- 媒体大小和时长限制；
- 临时文件使用随机名和受控目录；
- 防止路径穿越；
- 媒体定期清理；
- 外部图片下载限制重定向、超时和响应大小；
- 不执行媒体元数据中的命令或脚本。

## 8. 通知滥用保护

- Client 和插件限流；
- 每分钟、每小时发送上限；
- `@all` 单独限流；
- critical 和语音单独限流；
- 相同 event key 去重；
- 可选相似内容短期抑制；
- 持续催办有最短间隔、最大次数和截止时间；
- 插件突发大量事件时可聚合或暂停；
- 管理后台显示近期限流和抑制记录。

## 9. 数据库队列可靠性

### 原则

- API 接受事件前先完成持久化；
- 数据库事务中不执行网络请求；
- Worker claim 使用短事务；
- `processing` 有租约超时；
- Worker 崩溃后可回收；
- 每次尝试单独记录；
- 失败分为可重试和永久失败；
- 重试有上限；
- dead 可由管理员手工重新排队。

### SQLite 单实例约束

- 只运行一个 Notify Hub 实例；
- WAL 模式；
- busy timeout；
- 合理连接池；
- 避免长事务；
- 大批量清理分批执行；
- 定期备份前执行一致性检查。

## 10. 幂等与重复

平台提供“至少一次接收，效果上只发送一次”的目标。

核心机制：

- Event 唯一约束；
- 企业微信回调消息去重；
- Delivery 唯一约束；
- 插件游标在 emit 成功后推进；
- API 重试返回已有 event_id；
- Worker claim 防止并发重复处理；
- Provider duplicate check 可作为额外保护，不能代替本地幂等。

对于消息已发送但响应丢失的极端情况，可能发生重复投递。需要：

- 保存 provider request/response 标识；
- 使用企业微信支持的重复检查字段；
- 监控重复率；
- 不以“先标成功再发送”来避免重复，因为那会造成消息丢失。

## 11. 可用性与降级

### 企业微信不可用

- Event API 仍接受并排队；
- Delivery 进入 retry_wait；
- ready 不因单次渠道故障失败；
- 后台显示渠道 degraded；
- 恢复后继续发送。

### 插件来源不可用

- 插件记录失败；
- 不推进游标；
- 其他插件正常运行；
- 连续失败进入 degraded/failed；
- 可选发送“插件异常”平台通知，需防止告警风暴。

### 数据库不可用

- ready 失败；
- Event API 返回 503，不假装接受；
- Worker 停止 claim；
- live 仍可用于判断进程存活。

### ASR/TTS 不可用

- 文本通知继续；
- 语音输入提示失败；
- 语音输出降级文本；
- 不影响核心事件 API。

## 12. 日志和审计

结构化日志包含：

- timestamp；
- level；
- service；
- request_id；
- trace_id；
- actor_type/actor_id；
- event_id；
- delivery_id；
- plugin_id/plugin_run_id；
- error_type；
- message。

不得包含：

- API Key；
- Authorization；
- Cookie；
- 企业微信 Secret/Token；
- 完整回调密文；
- 解密后的原始 XML；
- 未脱敏手机号等个人信息；
- 完整外部响应正文。

审计日志至少覆盖：

- 登录；
- API Client 创建、轮换、吊销；
- Secret 更新；
- 插件启停和配置；
- 广播；
- dead 重试；
- 提醒取消和强制完成；
- 数据清理和备份恢复。

## 13. 备份与恢复

备份对象：

- SQLite 数据库；
- 加密 Secret 数据；
- 插件私有数据；
- 上传媒体中需要长期保留的部分；
- 配置版本信息。

主加密密钥必须单独备份。只有数据库而没有主密钥，Secret 无法恢复。

恢复演练：

1. 新建空目录；
2. 放入备份数据库和主密钥；
3. 使用同版本镜像启动；
4. 执行迁移；
5. 验证插件状态、API Client 和提醒；
6. 发送测试通知；
7. 检查待投递任务是否继续。

至少在首次正式使用前完成一次恢复演练。

## 14. 安全响应

怀疑泄露时：

1. 吊销相关 API Client；
2. 轮换企业微信 Secret；
3. 轮换平台 Session/JWT Key；
4. 暂停插件；
5. 检查审计日志和异常广播；
6. 备份现场数据；
7. 清除旧凭据并重新部署；
8. 必要时通知企业微信成员。

## 15. 发布前安全清单

- [ ] 没有默认管理员密码；
- [ ] 生产模式拒绝弱主密钥；
- [ ] 所有外部 API 鉴权；
- [ ] 企业微信回调验签和解密；
- [ ] Secret 不回显；
- [ ] 日志脱敏测试通过；
- [ ] `@all` 权限测试通过；
- [ ] 限流启用；
- [ ] SSRF 防护测试通过；
- [ ] 文件上传和路径穿越测试通过；
- [ ] Worker 崩溃恢复测试通过；
- [ ] 数据库备份和恢复演练完成；
- [ ] Docker 非 root；
- [ ] HTTPS 和反向代理配置正确；
- [ ] 依赖安全扫描无已知高危问题。

## 16. AI Gateway 安全边界

- Provider URL 禁止凭据和 fragment，默认只允许 HTTPS；HTTP 仅用于管理员显式允许的私网端点。
- DNS 解析结果与实际连接 peer 必须一致；云元数据地址始终禁止；Provider 重定向默认拒绝。
- API Key 使用 SecretStore，管理 API 只返回 configured 状态；调用日志不保存正文、Prompt、Authorization 或 Provider 响应正文。
- 外部内容被视为不可信数据，系统 Prompt 明确忽略其中的指令；模型不获得工具、通知能力或配置写权限。
- 结构化输出必须经 Pydantic 校验；不得从任意自然语言中猜测业务结论。
- 插件未获 Profile 权限、预算耗尽、超时或结构化输出失败时返回稳定错误；监控插件默认 fail-closed 且不推进当前游标。
