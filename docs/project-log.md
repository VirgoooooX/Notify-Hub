## 记录原则

- 只记总结，不记流水账。
- 按主题记录，条目日期精确到天，不按月份堆叠。
- 不记录 Token、Cookie、账号、私钥、真实敏感 URL 或 Secret 路径。
- commit hash 只作为回溯线索，少量保留。
- 详细设计与操作规范单独成文；本档案只保留结论和维护原则。

### 2026-07-16 - Reminder Center 完整系统

### 2026-07-16 - 企业微信图文与公共媒体能力

- 摘要：本地图片资产可生成带签名的公网 `picurl`；手机端和后台上传统一写入 `MediaAsset`。
- 影响：Codex 使用内置静态封面，Romano 下载并处理来源图片，提醒图文不再依赖外部热链。
- 结论：静态 URL 和签名媒体 URL 统一由 `PublicMediaUrlBuilder` 生成；没有独立跳转链接时使用封面 URL。
- 验证：公开媒体、WeCom news payload、提醒快照和两个监控插件测试通过。
- 相关提交：`3b32ce7`

## Debug / 复盘

### 2026-07-16 - 企业微信图文封面不显示

- 现象：提醒已保存本地图片资产，但企业微信 `news.picurl` 为空；无独立 URL 的图文还可能降级为文本。
- 结论：Adapter 只读取外部 `image_url`，没有把 `media_asset_id` 转成公网签名地址，并错误地把“无跳转链接”等同于“不能发送图文”。
- 处理：核心统一生成签名媒体 URL，并允许有媒体、无独立链接的 article 保持图文类型。
- 验证：后端相关测试 95 项、前端 27 项及 Ruff、Mypy、lint、typecheck、生产构建通过。

### 2026-07-16 - 全员完成通知重复与排版松散

- 现象：全员完成消息在标题和正文重复任务名，个人操作回执分段过多。
- 结论：模板重复承担状态强调，移动端显示显得零碎。
- 处理：全员完成收敛为单行；个人操作回执保留任务名并改为单行。
- 验证：菜单回调、失败重试、全员完成广播和幂等事件测试通过。

### 2026-07-15 - v0.5 插件与媒体校验失败

- 现象：模块化插件与媒体能力首次合并后出现配置、签名媒体和公共类型校验失败。
- 结论：共享 X 来源、插件 Context、公开媒体和发布脚本边界需要统一，而不能由插件各自兼容。
- 处理：收敛 Runtime Adapter、媒体签名和共享模型，并补 Runtime 与公开媒体测试。
- 相关提交：`da2a6b8`

### 2026-07-14 - 媒体包漏跟踪与安全依赖阻断

- 现象：媒体模块被 `.gitignore` 误排除，随后依赖审计和镜像扫描阻断发布。
- 结论：生成目录规则不能覆盖源码包；安全工具版本和排除策略需要明确固定。
- 处理：修复 Git 规则并补齐媒体模块，升级 cryptography、pytest 和 Trivy Action。
- 相关提交：`2e5d14f`, `08cf7d8`, `cc11bb6`

### 2026-07-14 - 本地 Secret 与前端格式问题

- 现象：插件无法正确读取本地 `.env` Secret，Ruff、ESLint 和 Vue 模板格式又连续阻断发布。
- 结论：Secret 来源解析和格式门禁必须在本地开发与 CI 中保持一致。
- 处理：补本地环境变量解析并修复格式告警。
- 相关提交：`83a7db0`, `5a8aa5c`, `c492463`
